Rapid Ideation Topic

For my first rapid ideation the topic was randomly selected by drawing a Dixit card live on video webinar. The card that was chosen was a picture of a blue dragon and a child with a sword which is shown below. I was happy that this card was chosen because as a games developer a fantasy theme can cater to a wide variety of creative ideas. Being a practical person, I wanted to make a game that was more literal than metaphorical, so I decided to make a video game that featured dragons and war as I thought the child with a sword represented war.

The dixit card that was revealed live on stream for the first rapid ideation project topic (Falmouth University, 2021).

My first idea was to create a pixel art platformer game where you defeat dragons as you progress through the level, but I decided against it because despite the platformer genre not being that difficult to develop the environment art would have taken too much time to create, I needed a level that was small and was static. Regardless, generally it is advised to not go with your first idea when developing a game for a games jam as excitement usually clouds judgement of the idea that is going to be created and more thought must be put into it to get the right scale of scope and feasibility.

Project Management

For my game projects I will adopt an agile project management style. To define agile “Agile methodology is based on iterative development with which requirements and solutions evolve through collaborative effort.” (Stern, 2017) and there are several benefits to using this methodology. To highlight a few examples of why agile is beneficial for video game development, firstly, agile projects finish on time which benefits teams that struggle with time constraints. Secondly, agile projects deliver high quality products results without inefficiencies or bugs in code. Lastly, code built by agile teams is well constructed and maintainable (Stellman and Greene, 2014).

For my first game project I will use the Kanban methodology using the website Trello. To define Kanban, “Kanban is a method for process improvement used by agile teams. Teams that use Kanban start by understanding the way that they currently build software, and make improvements to it over time.” (Stellman and Greene, 2014) and I will use Trello to manage the various mechanics I will have to write and implement and bullet point any improvements or suggestions I can make to existing mechanics or for future consideration.

Kanban agile methodology illustration (Maria Thomas, 2020).

Prototype: Design

To help me come up with an idea for my first rapid ideation I created a mind map to decide on a genre of video game I wanted to make. I came up with a few genres that would be suitable and non-suitable regarding scope and out of the suitable genres were a tower defense game, platformer, and flight shooter. I decided on a tower defense game for a couple of reasons, firstly because I can create a level that is small and static, the other genres will require a dynamic background, and this would take too much time on the development process and secondly, I haven’t had experience with creating a 2D tower defense game and it would be beneficial for me as a programmer to write some original scripts to know how the genre operates. I also decided to create it for android devices and PC to get more experience with developing on these platforms.

Mind map that I created to help generate ideas for my first rapid ideation project. 

I have also created a mood board that shows several images relating to a tower defense game with a fantasy theme to assist in generating ideas. I have included different images of dragons, kingdoms, existing tower defense games both 2d and 3d and a portal that could represent where the enemies could spawn from. 

Mood board that I created to come up with the theme of the tower defense game. Images by: (SergiuHellDragoonHQ, 2020), (PC Games N, 2021), (Alexandro Lima, 2016), (HZW, 2021), (Martin, 2019), (Renall, 2016), (Jean-Luc De Gruchy, 2021), (2nd Studio, 2018), (Ess Games, 2021)

Prototype: Development

To start, I set up a Trello board to manage the various tasks while I develop the game. I have chosen Trello to manage my workflow as I prefer the Kanban methodology as it allows for faster development and clarity regarding what needs to be done in the development process.

The trello kanban board that I setup to manage my project.

I have created some placeholder assets for the game that includes dragon enemy variants, tower variants, user interface icons, projectiles, and environment art. I have created these in Aseprite a program that is used to create art in pixel art style. I have chosen the pixel art style as in my opinion it’s the easiest form of art that even a programmer who is an inexperienced artist can accomplish and creating these assets would improve my art skills needed for further games art asset development. I’ve also created a placeholder environment that has a tileable background that will serve as the static level for where I can begin the development of the tower defense game.

Placeholder gameplay assets created in Aseprite.

Placeholder environment assets created in Asprite that serves as the level environment and a tileable sprite to cover any out of bounds space to cater to all aspect ratios.

I decided to develop the game in Unity as when compared to Unreal Engine, generally Unity has a larger focus on its 2D toolsets and on the other hand Unreal has greater 3D capabilities and graphical fidelity (Buckley, 2021). But since I am creating a 2D game I have decided to use this game engine instead of Unreal. Regardless, although I would like to improve my skillset in the Unreal Engine and in the C++ programming language, I don’t have enough time to learn it and develop a competent prototype before the deadline.

Unity vs Unreal Engine 4 (PluralSight, 2015).

Afterwards I created private git repository for my game using GitHub Desktop a source control software that is used for project file management. I have set up my Unity Project and the environment then made my repository and pushed my first commit. I gave my project the name ‘Once Upon a Fantasy’ to represent the theme of the game.

My first commit for Once Upon a Fantasy, 2D tower defense game.

After setting up an enemy spawn system I had a few options of how I could approach the pathfinding using algorithms such as A star or breath/depth first search which could have all taken time to research and implement but I opted to take an easier route of moving the enemies through several waypoints instead using an update function until they eventually reach the player base and get destroyed. From this I learned that the most popular solutions aren’t necessarily the most efficient ones and simple solutions might save development time.

Enemy spawns move and follow waypoints.

I wrote a build menu which is used to create the tower bases and weapons. I decided to have the menu appear next to the tower foundation and flip depending on where it is located on the screen as sometimes it will appear out of the boundaries. This made me learn a valuable lesson about keeping user interface elements in the screen boundaries especially on mobile devices.

The first iteration of my build menu.

I used scriptable objects to store data such as tower bases, weapons, projectiles, and levels then I wrote a public database where I could store these elements. When programming, creating scriptable objects and public databases are standard in my methodology as it is an efficient way of storing data and reading from it so that I don’t have to manually write out new objects in code from scratch which would take up unnecessary time.

My scriptable objects that I use to store data.

My various databases that I put my scriptable objects in to use in code.

I have also written a ‘Game Manager’ class and that stores all game relevant information such as the current level, level database, player resources, base health, level music and others. I have made this class for ease of level management and in the process, it has made me more code efficient.

My game manager class that is attached to a system game object.
