I watched two videos on the Waterfall and Scrum Agile Framework methodologies. I have learned that waterfall is a linear based project management methodology which relies heavily on documentation and planning before development starts. Very little user feedback is collected until the end and the methodology relies on taking the risk that the project will be at acceptable standards in the end. Agile on the other hand is an iterative approach to project management which relies on user input and feedback which is collated and collaborated on in Sprints and regular meet ups with the team, this results in the end project being better in quality. Week 3 - Rapid Ideation

Part 1

This week on my MA I will focus on ‘rapid ideation’ in relation to video games in preparation for the game jam in the following week. There are two rapid thinking processes for video games which are game jams and hackathons. Games jams and hackathons are generally the same kind of event where people come together and collaborate to create working software or hardware. Games jams usually have no pressure involved but hackathons are usually in the form of a competition. The result is the same for both events, to develop a new innovative product. There are techniques that will aid in the productivity and idea generation of a rapid ideation event and by the end of this week I will put these techniques into practice ready for future game jam and hackathon events.

Prototype: Design

For the design of the Stacker prototype, I have created a digital storyboard of the stacker arcade game and how the gameplay mechanics work in general. The rules of the game are the player must stack blocks and reach the top of the grid without losing all of them. Depending on the position of the player the game will set the maximum number of blocks and movement speed. For my game this is called Classic Mode as it retains all the rules of the original arcade game but to put a twist on the classic mode, I have created a game mode called Endless where the player can endlessly stack blocks and increase a counter so they can achieve a high score. I have made this mode as its generally considered bad practice to copy a game in its entirety so to make my game unique I had to put at twist on the existing formula for players to find an interest in the game.

A digital storyboard showcasing the game mechanics of the Stacker arcade game.

Prototype: Development

Before being tasked with this challenge I had already created a main menu scene and have created the game board and in-game menu, which is everything apart from the gameplay mechanics.

The main menu and game board setup for the Dash and Stack game for android devices.

First, I had to make the blocks move and go out of bounds of the screen. I wrote a coroutine that fires a Raycast one block ahead of the first or last child depending on if the player is reversing or not, if the player hits a boundary layer they will reverse. I could have used triggers to track the collisions of the blocks, but I wanted to use a Raycast because firstly I don’t have much experience using Raycasts and it would be beneficial for my development process to get more experience with this element and secondly, I can control a Raycast more precisely than a trigger for example only wanting to gain hits in a specific direction rather than all around the block with a 2D box trigger. I also moved the boundaries out more so the player blocks can clip outside the grid like the arcade game as my main goal is realism.

Basic movement and reversing an the player block intentionally going out of bounds to promote realism.

Next, I achieved stacking of blocks by having the next spawned block become the active player block and have each block child do a Raycast beneath them to check if there is a block beneath them, if not they will fall to the ground using a coroutine and play a lose animation like the arcade game. To demonstrate how I utilized a Raycast the gif below will show the debug lines that the ray fires in order to perform hit checks. Again, I could have used triggers here attached to the block to check when it collides with another block, but I wanted to stay with using Raycasts, this way my code is cleaner and more consistent.

Player losing blocks and moving and falling raycast demonstration.

Prototype: Result

I have managed to fully recreate Stacker arcade using a grid, blocks, Raycasts and coroutines. I have replicated the rules and the animations for a classic mode with plans to create an endless mode to create a gameplay twist to differentiate it from the original game. Everything has gone well but if there is one element, I would like to change it would be to rebalance the difficulty, as it stands the speed has 3 settings, a slow speed when you have 3 blocks, a medium speed when you have 2 blocks and a fast speed when you have 1 block. I think that instead of having it this way the speed should gradually increase making the game increasingly difficult the more the player stacks a block. Also, due to time constraints I did not manage to match up the lives of the blocks to those on the game menu and this is another element that I would like to change going forward. Overall, the reconstruction went well but for next time I would like to make the game more difficult and make it more featureful.
